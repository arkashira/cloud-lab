3. **Invite generation** – `InviteMember` returns a 32‑character URL‑safe token, stores the token with an expiry (default 48 h), and sends an email containing the token link.  
4. **Invite acceptance** – Providing a valid, unexpired token adds the user to the team, returns HTTP 200 with the team ID, and invalid/expired tokens return HTTP 410 (Gone) or 400.  
5. **Dashboard access control** – Only authenticated users whose `team_id` matches the requested dashboard return HTTP 200; others receive HTTP 403.  
6. **Concurrency safety** – Simultaneous `CreateTeam` or `InviteMember` calls for the same team name/token do not corrupt the in‑memory store or DB (no duplicate rows, no lost invites).  
7. **Audit logging** – Each successful creation, invitation, and acceptance writes an audit entry with timestamp, actor ID, and operation type.
