## 1. Acceptance Criteria
1. **Version selection** – The API returns a list of supported Terraform versions (0.12‑1.5) and accepts any version from that list; an invalid version returns HTTP 400 with a clear error message.  
2. **Provisioning latency** – A successful workspace creation request returns HTTP 202 and the workspace becomes “ready” within **≤ 120 seconds** from request receipt.  
3. **User isolation** – Resources (container, filesystem, network namespace) created for a workspace are scoped to the authenticated `user_id`; no other user can list, access, or modify those resources.  
4. **Sample state pre‑load** – Upon workspace “ready”, the container contains a predefined `sample.tfstate` file at `/workspace/state/sample.tfstate`.  
5. **Reset capability** – A `POST /workspaces/{id}/reset` call restores the sample state file to its original contents and returns HTTP 200 within **≤ 5 seconds**.  
6. **Idempotent creation** – Re‑submitting the same payload (same `user_id` + `workspace_name`) returns HTTP 200 with the existing workspace ID, without spawning a new container.  
7. **Observability** – Each provisioning request logs: request ID, user ID, selected version, start timestamp, end timestamp, and final status (success/failure).
