## Final Design – “Ansible‑Execution Service”

Below is a single, production‑ready implementation that pulls the strongest ideas from the two candidates:

* **Clear separation of concerns** – the service only orchestrates the run, while a pluggable `LogStore` handles persistence.
* **Robust temporary‑file handling** – every temporary file or directory is created in a safe, unique location and guaranteed to be removed even if the run panics.
* **Context‑aware execution** – the `exec.CommandContext` call respects the caller’s deadline or cancellation.
* **Explicit error reporting** – non‑zero exit codes, I/O errors, and log‑store failures are all surfaced to the caller.
* **Retention policy** – the `LogStore` interface exposes a `CleanupOlderThan` method so the caller can enforce a 7‑day (or any) policy.
* **Testability** – all external dependencies are injected (`exec.Command`, `os.CreateTemp`, etc.) via small wrapper interfaces so unit tests can mock them.
