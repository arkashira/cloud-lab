# docs/quickstart.md
# Quick Start Guide

Welcome to **cloud‑lab**! This guide will walk you through the essential steps to get your first project up and running, troubleshoot common issues, and explore our template examples.

## Prerequisites

- A GitHub account  
- Docker Desktop (or any container runtime)  
- `kubectl` installed and configured for your cluster  
- `helm` installed  

## 1. Clone the Repository

```bash
git clone https://github.com/[username]/cloud-lab.git
cd cloud-lab
```

## 2. Provision Your Environment

For a quick local test environment, run:

```bash
make dev
```

This will spin up a local Kubernetes cluster using kind and deploy the core services.

For cloud provisioning, see the detailed instructions in **[Provisioning Guide](/docs/quickstart/provisioning.md)**.

## 3. Deploy a Sample Application

```bash
helm install myapp ./charts/myapp
```

## 4. Verify

```bash
kubectl get pods -n myapp
```

You should see the pods in `Running` state.

## Troubleshooting

- **Pods stuck in `Pending`** – check node capacity and resource limits.  
- **Helm install fails** – ensure the chart repository is up to date (`helm repo update`).  
- **Connection refused** – verify that the service is exposed on the correct port.

For more detailed troubleshooting, refer to the [Troubleshooting Guide](/docs/troubleshooting.md).

## Explore Templates

We provide a set of ready‑to‑use templates to bootstrap common patterns:

- [Python Flask](/templates/python-flask.md)  
- [Node.js Express](/templates/node-express.md)  
- [Go Microservice](/templates/go-micro.md)

Happy hacking!

# docs/quickstart/provisioning.md
# Provisioning Guide

This guide explains how to provision cloud resources for your cloud‑lab projects.

## 1. Choose a Cloud Provider

cloud‑lab currently supports AWS, GCP, and Azure. Pick one and set up the CLI credentials.

### AWS

```bash
aws configure
```

### GCP

```bash
gcloud init
```

### Azure

```bash
az login
```

## 2. Create a Kubernetes Cluster

Use the provider’s managed Kubernetes service:

- **AWS EKS** – `eksctl create cluster --name cloudlab --region us-east-1`  
- **GCP GKE** – `gcloud container clusters create cloudlab --zone us-central1-a`  
- **Azure AKS** – `az aks create --resource-group cloudlab --name cloudlab --node-count 3`

## 3. Install Helm and Add Repositories

```bash
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
helm repo add cloudlab https://charts.cloudlab.example.com
helm repo update
```

## 4. Deploy Core Services

```bash
helm install core cloudlab/core
```

## 5. Verify Deployment

```bash
kubectl get pods -n cloudlab-core
```

All pods should be in `Running` state.

## 6. Next Steps

- Deploy your first application using the templates in the `templates/` directory.  
- Configure CI/CD pipelines via the `ci/` folder.  
- Monitor logs with `kubectl logs`.

For advanced configuration, see the [Advanced Provisioning Guide](/docs/advanced-provisioning.md).
