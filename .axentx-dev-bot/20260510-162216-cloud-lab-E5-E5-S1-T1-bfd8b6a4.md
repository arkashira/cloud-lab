# Quick Start Guide

Welcome to the cloud-lab quick start guide! This guide will help you get up and running with basic provisioning and troubleshooting.

## Basic Provisioning

1. **Install Prerequisites**:
   - Ensure you have Python 3.8 or later installed.
   - Install the cloud-lab CLI tool:
     ```bash
     pip install cloud-lab-cli
     ```

2. **Configure Your Environment**:
   - Set up your API keys and credentials in the `~/.cloud-lab/config` file.
   - Example configuration:
     ```yaml
     api_key: your_api_key_here
     region: us-west-1
     ```

3. **Provision Your First Resource**:
   - Use the CLI to provision a basic resource:
     ```bash
     cloud-lab provision resource --type vm --name my-vm
     ```

## Troubleshooting

### Common Issues

- **Authentication Errors**:
  - Ensure your API key is correctly set in the configuration file.
  - Verify that your key has the necessary permissions.

- **Resource Provisioning Failures**:
  - Check the availability of resources in your selected region.
  - Ensure you have sufficient quotas for the resource type.

### Support

If you encounter issues that are not covered in this guide, please visit our [Support Page](https://support.cloud-lab.com) or contact our support team at support@cloud-lab.com.

## Template Examples

For more advanced use cases, refer to our template examples:
- [Basic VM Provisioning](templates/vm_basic.md)
- [Advanced Networking](templates/networking_advanced.md)
- [Database Setup](templates/database_setup.md)
