### 1. Acceptance criteria
- User can select a script and drag it to an existing folder.
- Upon dropping, the script is moved to the target folder in the UI.
- The system updates the backend database with the new folder location for the script.
- User can create a new folder by dragging a script to an empty space and confirming the action.
- The newly created folder appears in the UI with the script inside.
- The system logs the creation of a new folder and the movement of the script in the audit log.

### 2. Unit tests
```javascript
// Test moving script to existing folder
test('moveScriptToExistingFolder', () => {
    const script = { id: 'script1', name: 'script.js' };
    const folder = { id: 'folder1', name: 'scripts' };
    const result = Scripts.move(script, folder);
    expect(result).toBe(true);
    expect(Folders.contains(folder.id, script)).toBe(true);
});

// Test creating new folder while moving script
test('createNewFolderWhileMovingScript', () => {
    const script = { id: 'script2', name: 'newscript.js' };
    const newFolderName = 'new-folder';
    const result = Scripts.createNewFolderAndMove(script, newFolderName);
    expect(result).toBe(true);
    const newFolder = Folders.getByName(newFolderName);
    expect(newFolder).not.toBeNull();
    expect(Folders.contains(newFolder.id, script)).toBe(true);
});
```

- Happy Case 1: Dragging a script into an existing folder updates both the UI and the backend.

- Edge Case 1: Attempting to move a script to a non-existent folder fails gracefully with an error message.
- Edge Case 2: Dragging a script over a file instead of a folder does not trigger any action.
