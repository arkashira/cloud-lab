## 1. Acceptance Criteria  

- **AC1** – The API endpoint `/labs` accepts a `POST` request with a valid Ansible playbook file and returns a JSON payload containing a non‑empty `lab_id` string within 2 seconds.  
- **AC2** – The lab runner starts within 30 seconds of receiving the request and completes execution of the playbook within 3 minutes, returning a JSON status `{ "status": "success" }` or `{ "status": "failure", "error": "<msg>" }`.  
- **AC3** – A lab instance is automatically terminated after 24 hours unless the user calls `/labs/{lab_id}/destroy` earlier.  
- **AC4** – The `/labs/{lab_id}/destroy` endpoint immediately terminates the lab and returns `{ "status": "destroyed" }`.  
- **AC5** – The system rejects malformed playbooks with a `400 Bad Request` and an error message.  
- **AC6** – The system logs the start, completion, and destruction timestamps for each lab in the format `YYYY-MM-DDTHH:MM:SSZ`.
