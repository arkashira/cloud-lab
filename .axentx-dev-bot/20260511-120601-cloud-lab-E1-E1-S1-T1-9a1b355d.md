## dev — claude/llm-fallback-chain @ 2026-05-11T12:44:46.557585Z

Below is a single, production‑ready Terraform sandbox image that pulls the best ideas from both candidates:

* **Minimal, reproducible base** – `debian:bullseye-slim` (no `apt-key`, no `apt-add-repository`).
* **Explicit version pinning** – Terraform 1.9.0, AWS‑CLI v2.15.0.
* **Non‑root sandbox user** – prevents accidental privilege escalation.
* **Environment‑driven backend** – `TF_STATE_BUCKET`, `TF_STATE_KEY`, `TF_REGION` (with sane defaults).
* **Automatic cleanup** – optional `CLEANUP_AFTER_SECONDS` (default 86400 s) that runs `terraform destroy` on exit.
* **Robust error handling** – `set -euo pipefail` and explicit checks.
* **Small runtime image** – only the binaries that are needed.

```dockerfile
# ────────────────────────────────────────────────────────────────────────
# 1️⃣  Build stage – install Terraform & AWS‑CLI
# ────────────────────────────────────────────────────────────────────────
FROM debian:bullseye-slim AS builder

# Install build‑time packages
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        curl \
        unzip \
        ca-certificates \
        gnupg && \
    rm -rf /var/lib/apt/lists/*

# Terraform
ARG TERRAFORM_VERSION=1.9.0
RUN curl -fsSL https://releases.hashicorp.com/terraform/${TERRAFORM_VERSION}/terraform_${TERRAFORM_VERSION}_linux_amd64.zip -o terraform.zip && \
    unzip terraform.zip -d /usr/local/bin && \
    rm terraform.zip

# AWS CLI v2
ARG AWS_CLI_VERSION=2.15.0
RUN curl -fsSL https://awscli.amazonaws.com/awscli-exe-linux-x86_64-${AWS_CLI_VERSION}.zip -o awscliv2.zip && \
    unzip awscliv2.zip -d /tmp && \
    /tmp/aws/install && \
    rm -rf awscliv2.zip /tmp/aws

# ────────────────────────────────────────────────────────────────────────
# 2️⃣  Runtime stage – keep the image tiny
# ────────────────────────────────────────────────────────────────────────
FROM debian:bullseye-slim

# Copy binaries
COPY --from=builder /usr/local/bin/terraform /usr/local/bin/terraform
COPY --from=builder /usr/local/bin/aws /usr/local/bin/aws

# Install only what we need at runtime
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        ca-certificates \
        curl && \
    rm -rf /var/lib/apt/lists/*

# Create a non‑root sandbox user
RUN useradd -m -s /bin/bash sandbox && \
    mkdir -p /home/sandbox/workspace && \
    chown -R sandbox:sandbox /home/sandbox

# Work‑dir for the user
WORKDIR /home/sandbox/workspace

# Copy the entrypoint
COPY entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh

# Switch to sandbox user
USER sandbox

# Default command
ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
```

```bash
# ────────────────────────────────────────────────────────────────────────
# 3️⃣  entrypoint.sh – orchestrates Terraform
# ────────────────────────────────────────────────────────────────────────
#!/usr/bin/env bash
set -euo pipefail

# -----------------------------------------------------------------------
# 3.1  Configuration – environment variables
# -----------------------------------------------------------------------
: "${TF_STATE_BUCKET:?TF_STATE_BUCKET must be set}"
: "${TF_STATE_KEY:?TF_STATE_KEY must be set}"
: "${TF_REGION:=us-east-1}"
: "${CLEANUP_AFTER_SECONDS:=86400}"   # 24 h default

# -----------------------------------------------------------------------
# 3.2  Create backend configuration
# -----------------------------------------------------------------------
cat > backend.hcl <<EOF
terraform {
  backend "s3" {
    bucket         = "${TF_STATE_BUCKET}"
    key            = "${TF_STATE_KEY}"
    region         = "${TF_REGION}"
    encrypt        = true
    dynamodb_table = "${TF_STATE_BUCKET}-locks"
  }
}
EOF

# -----------------------------------------------------------------------
# 3.3  Initialise and apply
# -----------------------------------------------------------------------
echo "=== Terraform init ==="
terraform init -backend-config=backend.hcl

echo "=== Terraform apply ==="
