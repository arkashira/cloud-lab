# Synthesized Ansible Playbook Library for Cloud-Lab Post-Provisioning

## Overview

This solution provides a complete, production-ready Ansible playbook library designed to run after Terraform provisions infrastructure. It handles Docker installation, user configuration, and system hardening with idempotent, reusable playbooks.

---

## Directory Structure

```
/opt/axentx/cloud-lab/backend/ansible/
├── playbooks/
│   ├── site.yml              # Main entry point
│   ├── docker.yml            # Docker installation & configuration
│   ├── users.yml             # User management & SSH keys
│   └── provision.yml        # Complete post-Terraform workflow
├── inventory/
│   └── inventory.ini         # Dynamic inventory file
├── group_vars/
│   └── all.yml               # Global variables
└── roles/                    # Optional role-based structure
```

## Playbooks

### 1. Main Entry Point (site.yml)

```yaml
# /opt/axentx/cloud-lab/backend/ansible/playbooks/site.yml
---
# Main entry point playbook - orchestrates all configuration after Terraform
- name: Main provision workflow
  hosts: all
  become: yes
  serial: 1
  gather_facts: yes
  
  tasks:
    - name: Include Docker configuration
      import_playbook: docker.yml
    
    - name: Include user configuration
      import_playbook: users.yml
    
    - name: Display completion message
      debug:
        msg: "All post-provisioning tasks completed successfully"
```

### 2. Docker Installation & Configuration (docker.yml)

```yaml
# /opt/axentx/cloud-lab/backend/ansible/playbooks/docker.yml
---
# Docker installation and configuration playbook
- name: Install and configure Docker
  hosts: all
  become: yes
  gather_facts: yes
  
  vars:
    docker_packages:
      - docker-ce
      - docker-ce-cli
      - containerd.io
      - docker-compose-plugin
    docker_service_state: started
    docker_enabled: yes
    docker_users: []
  
  pre_tasks:
    - name: Verify OS family is supported
      fail:
        msg: "Unsupported OS family: {{ ansible_os_family }}"
      when: ansible_os_family not in ["Debian", "RedHat"]
  
  tasks:
    # === Debian/Ubuntu Setup ===
    - name: Update apt cache (Debian)
      ansible.builtin.apt:
        update_cache: yes
        cache_valid_time: 3600
      when: ansible_os_family == "Debian"
    
    - name: Install prerequisites (Debian)
      ansible.builtin.apt:
        name:
          - apt-transport-https
          - ca-certificates
          - curl
          - gnupg
          - lsb-release
        state: present
      when: ansible_os_family == "Debian"
    
    - name: Add Docker GPG key (Debian)
      ansible.builtin.apt_key:
        url: "https://download.docker.com/linux/{{ ansible_os_family | lower }}/gpg"
        state: present
      when: ansible_os_family == "Debian"
    
    - name: Add Docker repository (Debian)
      ansible.builtin.apt_repository:
        repo: "deb [arch=amd64] https://download.docker.com/linux/{{ ansible_os_family | lower }} {{ ansible_distribution_release }} stable"
        state: present
      when: ansible_os_family == "Debian"
    
    - name: Install Docker packages (Debian)
      ansible.builtin.apt:
        name: "{{ docker_packages }}"
        state: present
      when: ansible_os_family == "Debian"
    
    # === RedHat/CentOS Setup ===
