# Azure App Services Tutorial 1 – Getting Started
## Overview
In this tutorial you’ll learn how to create your first Azure App Service, deploy a simple web app, and understand the core concepts that make Azure App Services a powerful platform for modern web applications.

### Prerequisites
- An Azure subscription (free trial works)
- Azure CLI installed (`az` command)
- Basic knowledge of HTML/CSS/JavaScript

### 1. Create a Resource Group
```bash
az group create --name MyAppServiceRG --location eastus
```

### 2. Create an App Service Plan
```bash
az appservice plan create --name MyAppServicePlan \
  --resource-group MyAppServiceRG --sku B1 --is-linux
```

### 3. Create a Web App
```bash
az webapp create --name my-unique-app-name \
  --resource-group MyAppServiceRG --plan MyAppServicePlan --runtime "NODE|14-lts"
```

### 4. Deploy a Sample App
Create a simple `index.html` in a folder and deploy using FTP or Git. For Git:
```bash
cd my-sample-app
git init
git remote add azure https://<your-azure-git-url>
git add .
git commit -m "Initial commit"
git push azure master
```

### 5. Verify Deployment
Open `https://my-unique-app-name.azurewebsites.net` in a browser. You should see your sample page.

### 6. Next Steps
- Explore scaling options
- Configure custom domains
- Set up continuous deployment

# Azure App Services Tutorial 2 – Configuring Environment Variables
## Overview
Environment variables let you store configuration data separately from your code. This tutorial covers how to set, view, and use environment variables in Azure App Services.

### 1. Set an Environment Variable via CLI
```bash
az webapp config appsettings set --name my-unique-app-name \
  --resource-group MyAppServiceRG --settings MY_SECRET_KEY=supersecret
```

### 2. View Current Settings
```bash
az webapp config appsettings list --name my-unique-app-name \
  --resource-group MyAppServiceRG
```

### 3. Use in Your App
In Node.js:
```js
const secret = process.env.MY_SECRET_KEY;
console.log(`Secret: ${secret}`);
```

### 4. Update or Remove
```bash
# Update
az webapp config appsettings set --name my-unique-app-name \
  --resource-group MyAppServiceRG --settings MY_SECRET_KEY=newvalue

# Remove
az webapp config appsettings delete --name my-unique-app-name \
  --resource-group MyAppServiceRG --setting-names MY_SECRET_KEY
```

### 5. Best Practices
- Never commit secrets to source control.
- Use Azure Key Vault for sensitive data.
- Rotate keys regularly.

# Azure App Services Tutorial 3 – Scaling and Performance
## Overview
Scale your app to handle traffic spikes and optimize performance. This guide covers manual scaling, autoscale rules, and performance monitoring.

### 1. Manual Scaling
```bash
az appservice plan update --name MyAppServicePlan \
  --resource-group MyAppServiceRG --sku P1v2
```

### 2. Autoscale Configuration
Create an autoscale rule:
```bash
az monitor autoscale create --resource-group MyAppServiceRG \
  --resource MyAppServicePlan --resource-type Microsoft.Web/serverfarms \
  --name MyAutoScale --min-count 1 --max-count 5 --count 1
```
Add a rule:
```bash
az monitor autoscale rule create --resource-group MyAppServiceRG \
  --autoscale-name MyAutoScale --scale out 1 --cooldown 5m \
  --condition "Percentage CPU > 70 avg 5m"
```
