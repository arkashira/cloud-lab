## 1. Acceptance Criteria  
- **Form Rendering**: The `/opt/axentx/cloud-lab/feedback/tutorial_feedback_form.html` page must load without JavaScript errors and display a rating widget (1–5 stars) and a text area for comments.  
- **Input Validation**:  
  - Rating must be an integer between 1 and 5.  
  - Comment length must be ≤ 500 characters.  
  - Both fields are required; form submission should block if validation fails.  
- **Submission Flow**: On submit, the form sends a POST request to `/api/feedback/tutorial` with JSON payload `{rating: int, comment: string, tutorialId: string}`.  
- **Server Response**:  
  - Success → HTTP 200 with JSON `{status: "ok"}`.  
  - Validation error → HTTP 400 with JSON `{status: "error", message: string}`.  
- **Data Persistence**: Feedback records are stored in the database table `tutorial_feedback` with columns `id`, `tutorial_id`, `rating`, `comment`, `created_at`.  
- **Review Trigger**: A background job runs nightly to aggregate feedback and updates the `tutorial_metrics` table; the job must log a summary of average rating and total comments per tutorial.
