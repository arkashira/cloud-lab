## 1. Acceptance Criteria
- **Step Count**: The tutorial page must expose exactly **5** sequential steps, each with a unique identifier (`step-1` … `step-5`).
- **Run Button**: Every step must render a **Run** button that, when clicked, triggers a POST to `/api/tutorial/execute` with the step number and the exact Azure CLI command for that step.
- **Command Execution**: The backend must execute the received Azure CLI command, capture its stdout/stderr, and return a JSON payload containing:
  - `status` (`"success"` or `"error"`)
  - `output` (full command output)
  - `step` (the step number)
- **Progress Bar**: The frontend must update a progress bar to **20%** increments per successful step completion. The backend should return the current progress percentage in the response.
- **Live App View**: After step 5 succeeds, the frontend must expose a link (`/app`) that opens the deployed Node.js web app. The backend should verify the app is reachable (HTTP 200) before confirming success.
